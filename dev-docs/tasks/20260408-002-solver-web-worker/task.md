# Task

## タスク種別
- refactor / feature

## 背景
- BFS ソルバーは重い計算（道を開けろ: ~5秒）で、メインスレッドで実行すると UI がフリーズする
- Phase 3 のヒント機能・解法リプレイでは、ゲームプレイ中にソルバーを実行する必要がある

## 目的
- Solver を Web Worker で非同期実行できるようにし、UI スレッドのブロックを防ぐ

## やること
- [ ] Solver を Web Worker 内で実行する仕組みを作成
- [ ] メインスレッドからの呼び出し用ラッパー（Promise ベース）を作成
- [ ] 任意の盤面状態からの求解に対応（ゲーム途中からのヒント用）
- [ ] 既存テストの互換性維持（Solver.ts 自体は変更しない）

## やらないこと
- UI の変更（ヒントボタン等は別タスク）
- Solver アルゴリズムの改善

## 完了条件
- [ ] Web Worker 経由でソルバーが動作し、結果が Promise で返る
- [ ] メインスレッドが計算中にブロックされない
- [ ] 既存テスト全パス + ビルド成功

## 制約
- Vite 6 の Web Worker サポートを利用する（`new Worker(new URL(...), { type: 'module' })`）
- 既存の Solver.ts のインターフェースは変更しない

## 参考ファイル
- `src/game/Solver.ts`
- `vite.config.ts`
