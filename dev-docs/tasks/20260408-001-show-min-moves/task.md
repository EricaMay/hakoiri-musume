# Task

## タスク種別
- feature

## 背景
- クリア画面では最適手数と比較表示があるが、ゲームプレイ中は自分の手数しか見えない
- プレイ中に目標を意識できると攻略のモチベーションが上がる

## 目的
- ゲーム画面のヘッダーに最適手数（`_minMoves`）を常時表示する

## やること
- [ ] `PuzzleDef` 型に `_minMoves` フィールドを正式追加
- [ ] ゲームヘッダーに「最短: XX手」を表示
- [ ] 既存テストが引き続きパスすることを確認

## やらないこと
- Solver をリアルタイムで実行して算出する（既存の静的データを使う）
- クリア画面の表示変更

## 完了条件
- [ ] ゲームプレイ中にヘッダーで最適手数が確認できる
- [ ] `_minMoves` がないパズルでは表示をスキップする
- [ ] typecheck, lint, unit test パス

## 制約
- 既存の UI レイアウトを大きく崩さない

## 参考ファイル
- `src/game/types.ts`
- `src/ui/ScreenManager.ts`
- `src/data/puzzles.json`
